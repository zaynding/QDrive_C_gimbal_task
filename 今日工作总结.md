### 4. 云台状态机

```text
CALIBRATING
-> RUNNING
-> RECOVERING / FAULT
```

退出 `RUNNING` 的条件：

- 任一电机超过 `50 ms` 没有反馈。
- 任一电机使能位变为 0。
- IMU 超过 `10 ms` 没有更新。
- 控制被停止。

视觉丢失本身不会退出 `RUNNING`。

### 5. 激光逻辑

- `RUNNING` 且视觉 `valid=1`、数据不超过 `100 ms` 时自动开激光。
- 目标无效或超时自动关激光。
- `RUNNING` 期间按键无效。
- 退出 `RUNNING` 后按键重新有效。

## 今天定位的核心故障

实机日志为：

```text
EXIT:IMU_OFFLINE PHASE=0 SPI=5 ERR=0x00000010 DRDY=1 AGE=11
```

含义：

- HAL SPI 仍处于 `BUSY_TX_RX`。
- 错误码是 `HAL_SPI_ERROR_DMA`。
- BMI 状态机却已经认为 DMA 空闲。
- 两套状态不一致，导致后续 DMA 一直无法启动。

最初增加的 HAL 错误回调只能释放片选，不能解决 HAL 内部忙状态，因此第一次修复不完整。

## 最新修复

已经严格参考示例工程的 `IMU_Task.cpp`，将 BMI088 采样改为纯 C 裸机直接 DMA：

- 不再使用 `HAL_SPI_TransmitReceive_DMA()`。
- 直接停止 DMA 流、清除 `TC/HT/TE/DME/FE` 标志。
- 直接设置 DMA 地址和长度。
- `DMA2_Stream0_IRQHandler` 直接处理 BMI088。
- Gyro 完成后启动 Accel。
- Accel 完成后解算姿态并通知裸机控制任务。
- DMA 错误时释放片选、清标志并回到 `IDLE`。
- 没有引入任何 FreeRTOS 代码。

最新 Keil 构建：

```text
0 Error(s), 0 Warning(s)
```

这个版本尚需烧录进行长时间实机验证。

## 仍未完成的问题

### 1. 无视觉时没有真正自稳

目前无视觉分支会：

```text
目标角对齐当前反馈
两轴输出 0 rpm
清空 PID
```

因此车体转动时云台会跟着车转。它符合当前代码，但不符合真正自稳。最终应让自稳与视觉解耦：

```text
无视觉：保持惯性空间目标角，继续自稳，激光关闭
有视觉：视觉更新目标角，继续自稳并追踪
```

### 2. 丢失目标后的慢速搜索尚未实现

当前目标丢失超过 `100 ms` 后停止追踪，不会扫描。之前讨论的方向是：

- 上电从未见过目标时保持静止。
- 曾经追踪过目标后，丢失时 Yaw 慢速往返搜索。
- Pitch 保持。
- 搜索范围不超过上电零点 `+/-30 deg`。
- 重新出现 `valid=1` 时立即恢复追踪。

### 3. 最新直接 DMA 版本需要验证

需要继续观察是否仍出现：

```text
EXIT:IMU_OFFLINE ...
```

## 对需求的理解

最终目标不是泛化云台框架，而是一套可以直接联调和参赛的激光追踪系统：

- 上电不能乱转。
- 树莓派数据随时都能重新接收并响应。
- 视觉追踪方向正确。
- 找到目标自动打激光。
- 目标丢失后不能永久退出，应能慢速重新寻找。
- 车体运动时云台能够自稳。
- 所有实现保持 C 语言、裸机、简单并贴近例程。

今天过程中曾把诊断版本和启动异常直接建立因果关系，提前撤回了诊断代码，这个判断不严谨。后来通过逐级增加只读日志，最终拿到了 DMA 错误码并定位到 HAL SPI 状态冲突。后续继续按照“先拿到证据，再做单点修改”的方式处理。
